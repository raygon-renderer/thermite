Expand description

Customizable Policy Parameters

Fields

check_overflow: bool

If true, methods will check for infinity/NaN/invalid domain issues and give a well-formed standard result.

If false, all of that work is avoided, and the result is undefined in those cases. Garbage in, garbage out.

However, those checks can be expensive.

unroll_loops: bool

If true, unrolled and optimized versions of some algorithms will be used. These can be much faster than the linear variants. If code size is important, this will improve codegen when used with opt-level=z

precision: PrecisionPolicy

Controls if precision should be emphasized or de-emphasized.

avoid_branching: bool

If true, methods will not try to avoid extra work by branching. Some of the internal branches are expensive, but branchless may be desired in some cases, such as minimizing code size.

max_series_iterations: usize

Some special functions require many, many iterations of a function to converge on an accurate result. This parameter controls the maximum iterations allowed. Setting this too low may result in loss of precision.

Note that this is the upper limit allowed for pathological cases, and many loops will terminate dynamically before this.
