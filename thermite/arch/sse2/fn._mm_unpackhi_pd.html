pub unsafe fn _mm_unpackhi_pd(a: __m128d, b: __m128d) -> __m128d
This is supported with target feature sse2 only.

The resulting __m128d element is composed by the low-order values of the two __m128d interleaved input elements, i.e.:

  • The [127:64] bits are copied from the [127:64] bits of the second input * The [63:0] bits are copied from the [127:64] bits of the first input
